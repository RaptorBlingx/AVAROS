## 🎯 Objective

Build a complete voice-controlled system where OVOS commands control real devices through an API backend. This simulates how AVAROS will interact with manufacturing platforms like RENERYO.

**What you're proving:** OVOS can control external systems via voice, not just answer questions.

---

## 📋 Prerequisites

✅ Weather skill task completed  
✅ Understanding of OVOS intents, dialogs, and API calls

---

## 🏗️ What You're Building

### Two Components:

1. **Mock Device API** (Backend)
   - Simulates an IoT/manufacturing device management platform
   - Manages state of multiple devices (machines, sensors, equipment)
   - Responds to control commands (on/off/status queries)

2. **OVOS Device Control Skill** (Frontend)
   - Voice commands to query device status
   - Voice commands to control devices (turn on/off)
   - Real-time feedback via spoken responses

---

## 🎤 Required Voice Interactions

Your system must support these conversations:

```
User: "Hey Mycroft"
OVOS: [beep]

User: "What's the status of device 1?"
OVOS: "Device 1 is online and currently running"

---

User: "Hey Mycroft, turn off machine 3"
OVOS: "Turning off machine 3... Done. Machine 3 is now off"

---

User: "Hey Mycroft, turn on light 2"  
OVOS: "Turning on light 2... Done. Light 2 is now on"

---

User: "Hey Mycroft, list all devices"
OVOS: "You have 5 devices: device 1 is on, machine 3 is off, light 2 is on, sensor 4 is online, and device 5 is in maintenance"
```

---

## 📊 Mock Devices Requirements

Your API should simulate **at least 5 devices** with generic names:
- Device 1, Device 2, ...
- Machine 1, Machine 2, ...
- Light 1, Light 2, ...
- Sensor 1, Sensor 2, ...

Each device should have:
- Unique ID (e.g., `device-1`, `machine-3`, `light-2`)
- Status states (online, offline, maintenance)
- Power states (on, off)
- Realistic metrics (power consumption, temperature, operating hours, etc.)

**Example device:**
```
Device 1:
- Status: online
- Power: on  
- Metrics: 45.2 kW consumption, 68°C temperature, 1248 operating hours
```

## 🎯 Objective

Build a document grounding system where OVOS answers questions about **ISO 50001 Energy Management Systems** by retrieving relevant information from indexed documents. This ensures AI responses are **grounded in factual documentation** rather than hallucinated.

**What you're proving:** AI can provide accurate, traceable answers backed by real documentation sources.

---

## 📋 Prerequisites

✅ Voice-Controlled Device Management System completed  
✅ Basic understanding of RAG (Retrieval-Augmented Generation)  
✅ Familiarity with document processing and semantic search

---

## 🎤 Required Voice Interactions

Your system must support these conversations:

```
User: "Hey Mycroft"
OVOS: [beep]

User: "What are the requirements for energy baseline?"
OVOS: "According to ISO 50001 section 4.4.3, the energy baseline 
       must be established using data covering an appropriate period. 
       Source: ISO 50001, section 4.4.3"

---

User: "Hey Mycroft, how do I conduct an energy review?"
OVOS: "The energy review should identify significant energy uses 
       and opportunities for improvement. Full requirements are in 
       ISO 50001 clause 4.4.3. Source: ISO 50001, page 12"

---

User: "Hey Mycroft, what is EnPI?"
OVOS: "EnPI stands for Energy Performance Indicator. It's used to 
       quantify energy performance as defined in ISO 50001 section 4.4.5. 
       Source: ISO 50001, section 4.4.5"

---

User: "Hey Mycroft, help with management review"
OVOS: "Management review requirements are covered in ISO 50001 section 4.6. 
       The review must evaluate opportunities for improvement and 
       the need for changes to the energy management system. 
       Source: ISO 50001, section 4.6"
```

---

## 📚 Document Set Requirements

Use **ISO 50001 Energy Management Systems** documentation as your corpus:

**Required documents:**
- ISO 50001 standard (PDF/text) - primary source
- ISO 50001 implementation guides (if available)
- Energy management procedure templates
- Related energy management documentation

**Minimum:** At least 1-2 substantial documents (50+ pages total)

**Why ISO 50001?**  
Real-world technical document with complex structure - perfect for testing semantic search and document grounding patterns.
